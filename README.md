# 公交站台设施维护管理系统

一个全栈的公交站台设施维护管理系统，支持站台管理、巡检任务、维修工单和统计后台。

## 技术栈

- **前端**：Vite + React 18 + TypeScript（默认端口 5176）
- **后端**：Express + TypeScript + better-sqlite3（默认端口 3204）
- **认证**：JWT + bcryptjs
- **启动**：concurrently 并行启动前后端
- **代理**：前端 Vite 代理 `/api` 和 `/uploads` 到 `http://localhost:3204`

## 启动方式

```bash
# 安装依赖
npm run install:all

# 启动项目
npm run dev
```

启动后访问 http://localhost:5176

## 功能说明

### 站台管理
- 管理员录入公交站台信息（名称、位置、所属线路、设施清单）
- 设施类型：候车亭、座椅、站牌、照明、遮雨棚
- 站台列表支持区域筛选和名称搜索，每个站台卡片显示设施完好率

### 巡检任务
- 巡检员查看今日巡检任务列表
- 逐项检查设施状态：完好、损坏、缺失
- 损坏设施可拍照上传并填写损坏描述
- 提交后自动生成维修工单

### 维修工单
- 维修员查看待办工单列表
- 处理工单：上传修复照片、标记完成
- 工单支持状态流转：待处理 → 处理中 → 已完成

### 统计后台（管理员）
- 各站台设施完好率排行
- 维修工单处理及时率
- 月度巡检完成率

## 注意事项

1. 首次启动会自动创建数据库、建表并 seed 预置数据
2. 数据库文件位于 `server/database.db`
3. 上传的照片存储在 `server/uploads/` 目录
4. 样式全部写在 `index.html` 的 `<style>` 标签中，未使用 CSS 文件或第三方 UI 库
5. JWT Token 有效期为 24 小时
6. 可通过 `VITE_PORT`、`VITE_API_PORT`、`PORT` 覆盖默认端口
